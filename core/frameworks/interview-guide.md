# 客戶需求訪談指南（§0 + §0.5）

> 每份分析報告開始前，先偵察 Dashboard，再透過 7 個問題釐清客戶需求。
> 若客戶未回答，使用預設值快速推進。

---

## §0.5: Dashboard 偵察（訪談前必做）

> **在問任何訪談問題之前，先打開 Dashboard 掃一遍。**
> 這樣訪談時才能給出具體選項，而不是問出 Dashboard 不支援的問題。

### 偵察步驟

1. **開啟 Dashboard URL** — 確認可正常載入
2. **掃描所有頁面** — 記錄左側導覽欄的全部分頁名稱
3. **掃描篩選器可用品牌** — 打開主題篩選器，滾動查看所有可選品牌
4. **記錄可用品牌清單** — 用於 Q7 競品建議（不要建議 Dashboard 沒有的品牌）
5. **識別頁面類型** — 標記哪些頁面適用品牌分析、哪些不適用（如地圖評價）

### 偵察產出

```markdown
Dashboard 偵察結果：
- URL: [Dashboard URL]
- 可用頁面: [列出所有頁面]
- 可用品牌: [列出篩選器中的品牌]
- 品牌分析不適用頁面: [如：地圖評價分析（品牌非地標，資料無意義）]
- 篩選器類型: [標準7步驟 / 有特殊預設]
```

### 為什麼要先偵察

- 避免訪談時建議不存在的競品（如 Dashboard 沒有 Hermès）
- 提前知道有哪些頁面，訪談時能問「要不要深入分析 KOL 頁？」
- 識別特殊篩選器行為（如搜尋意圖頁預設全選 vs 社群頁預設單選）

## 七大訪談問題

### Q1: 報告給誰看？

- **為什麼重要：** 決定行動建議的語氣和用詞層級
- **選項範例：** 廣告主/品牌方、內部業務、政府客戶、投資方、媒體
- **影響：** 行動建議語氣
  - 品牌方 → 直接建議行銷操作
  - 政府客戶 → 偏政策建議、量化成效
  - 投資方 → 偏市場趨勢、品牌價值
  - 媒體 → 偏故事性、數據亮點

### Q2: 分析的主要目的？

- **為什麼重要：** 決定哪些維度需要深挖，哪些可以帶過
- **選項範例：** 業務提案、品牌健檢、競品比較、活動效果評估、年度報告
- **影響：** 深挖維度
  - 業務提案 → 強調市場機會、成長空間
  - 品牌健檢 → 全面均衡，14 維度盡量覆蓋
  - 競品比較 → 維度 9 為必做，需多品牌篩選
  - 活動效果評估 → 聚焦時間區間前後對比
  - 年度報告 → YoY 為必做，需完整時間軸

### Q3: 報告的核心論點？（可跳過）

- **為什麼重要：** 若客戶已有預設假設，優先驗證可節省時間
- **選項範例：** 「我們品牌在台灣聲量持續成長」、「代言人策略有效」
- **影響：** 有預設論點 → 優先用數據驗證或推翻；無預設 → 由數據引導論點

### Q4: 特別想知道，或擔心被問到的問題？

- **為什麼重要：** 識別需要深挖或提前準備防禦說法的區域
- **選項範例：** 「老闆會問為什麼 Q3 聲量掉」、「想知道 KOL 合作效果」
- **影響：** 標記為高優先分析區域，確保報告中有明確回應

### Q5: 行動建議的執行對象？

- **為什麼重要：** 行動建議的 WHO 欄位必須指向具體執行方
- **選項範例：** Stage101 業務、品牌 PR、客戶行銷部、百貨樓管
- **影響：** 決定行動建議中 WHO 欄位的填寫對象
  - Stage101 業務 → 建議偏合作提案、媒體資源
  - 品牌行銷部 → 建議偏內容策略、預算分配

### Q6: 報告格式 & 風格偏好？

- **為什麼重要：** 影響產出方式和視覺呈現
- **選項範例：**
  - 產出格式：PowerPoint 檔案 .pptx（預設，離線可用，適合正式提案）、Google Slides（線上可編輯，適合協作和即時分享）
  - 配色：品牌色（預設）、中性專業色、客戶指定色
  - 語言：繁體中文（預設）、英文、雙語
  - 長度：完整版 13 張（預設）、精簡版 5 張
- **影響：**
  - pptx → 使用 pptxgenjs 產出本地檔案（`ppt-template.md`）
  - Google Slides → 使用 Slides API 產出線上簡報（`slides-template.md`），首次需 OAuth 授權
  - 配色方案、文案語言、投影片張數

### Q7: 需要競品比較嗎？

- **為什麼重要：** Dashboard 篩選器支援多品牌切換，需提前確認
- **選項範例：** 不需要（預設）、需要比較 Chanel 和 Hermès
- **影響：**
  - 需要 → 維度 9「競品比較」升級為必做，Dashboard 需多輪篩選提取
  - 不需要 → 維度 9 跳過

## 確認模板

訪談結束後，用以下模板向客戶確認：

> 「好，這份報告的目標是 **[目的]**，給 **[受眾]** 看，核心論點是 **[方向]**，執行對象是 **[執行方]**，格式為 **[pptx / Google Slides]**，風格 **[配色/語言/長度]**，**[需要/不需要]** 比較競品 **[競品名]**。還有補充或修正嗎？」

### 範例

> 「好，這份報告的目標是 **品牌健檢**，給 **品牌方** 看，核心論點是 **台灣市場聲量趨勢**，執行對象是 **品牌行銷部**，格式為 **pptx**，風格 **品牌色/繁中/完整版 13 張**，**不需要** 比較競品。還有補充或修正嗎？」
