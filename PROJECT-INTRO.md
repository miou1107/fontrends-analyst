# Fontrends Data Analyst — AI 品牌分析助理

> 一套由 AI 驅動的品牌社群數據分析系統，從訪談、數據擷取、統計分析到報告產出，全程自動化。

---

## 這套系統在做什麼？

簡單來說：**把品牌的社群數據變成有洞察的分析報告**。

過去做一份品牌分析報告，需要分析師花 2-3 天：
- 手動從 Dashboard 抓數據
- 用 Excel 做交叉分析
- 寫報告、做簡報

現在，這套系統可以在 **幾分鐘內** 完成同樣的工作，而且品質更穩定。

| 角色 | 負責什麼 |
|------|---------|
| **管理者** | 統籌全局，決定流程順序，分配任務 |
| **訪談員** | 跟你一問一答了解品牌需求（想分析什麼？競品是誰？） |
| **研究員** | 自動蒐集搜尋趨勢、競品數據、新聞事件 |

### 2. 從頭到尾自動化的分析流水線

整個分析過程分 8 個階段，每個階段的產出自動流入下一階段：

![品牌分析 Pipeline 流程](https://s3-alpha.figma.com/thumbnails/a3ea0ba0-d413-4843-9143-5566c50ed690?X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Credential=AKIAQ4GOSFWC6RGVDPLF%2F20260324%2Fus-west-2%2Fs3%2Faws4_request&X-Amz-Date=20260324T063744Z&X-Amz-Expires=604800&X-Amz-SignedHeaders=host&X-Amz-Signature=33e03b9f6a56c8d68f58c285b4f9aa65e5dcb2cbb15ed151a499c8b892254bf0)

**流程說明：**

| 階段 | 做什麼 | 產出 |
|------|--------|------|
| **1. 訪談** | AI 問你品牌背景、分析重點、競品是誰 | interview.json |
| **2. 研究蒐集** | 自動查搜尋趨勢、關鍵字、新聞事件 | research.json |
| **3. 數據擷取** | 從資料庫撈出社群聲量、平台分布、KOL 排行等 | data.json |
| **4-6. 數據分析** | 趨勢分析、交叉驗證、異常偵測 | analysis.json |
| **7. 敘事包裝** | 把數據變成有故事的章節內容 | narrative.json |
| **9. 報告產出** | 自動生成 Google Docs / Slides / PPT | 完成的報告 |
| **10. 品質審核** | 6 個維度打分，找出需要改進的地方 | audit.json |
| **12. 自我學習** | 記錄每次分析的經驗，下次做得更好 | 更新技能 |

### 3. 多層數據來源，確保分析品質

系統不依賴單一數據源，而是分層取用：

![數據來源分層](https://s3-alpha.figma.com/thumbnails/8e07fc08-8bc2-4b5f-88e8-16303dbe00d3?X-Amz-Algorithm=AWS4-HMAC-SHA256&X-Amz-Credential=AKIAQ4GOSFWC6RGVDPLF%2F20260324%2Fus-west-2%2Fs3%2Faws4_request&X-Amz-Date=20260324T063814Z&X-Amz-Expires=604800&X-Amz-SignedHeaders=host&X-Amz-Signature=8f1cb99f43b28e2ea2c94b16b7826353489eca51d672a1c40b83b23d9810381b)

- **資料庫**（免費、最快）：574 萬筆社群文章 + 77 萬筆搜尋數據
- **DataForSEO API**（付費補充）：即時新聞、競品搜量、關鍵字建議
- **Looker Studio**（視覺化確認）：Dashboard 截圖、交叉驗證

### 4. 內建完整的分析方法論

系統不是隨便跑數字，背後有一套 **16 章節的分析方法論**，涵蓋：

```
┌─────────────────────────────────────────────────┐
│                 分析方法論核心                      │
├─────────────────────────────────────────────────┤
│                                                 │
│  影響力傳導鏈                                     │
│  社群聲量 → 搜尋量 → 網站流量 → 銷售/到訪           │
│                                                 │
│  三層分析框架                                     │
│  ┌───────────┐ ┌───────────┐ ┌───────────┐     │
│  │ L1 趨勢分析│ │ L2 交叉驗證│ │ L3 歸因分析│     │
│  │ 找異常波動 │ │ 驗證因果鏈 │ │ 找外部原因 │     │
│  └───────────┘ └───────────┘ └───────────┘     │
│                                                 │
│  三層對比                                        │
│  ┌────┐  ┌────┐  ┌────┐                        │
│  │自比│  │競比│  │環比│                           │
│  │跟自│  │跟競│  │跟市│                           │
│  │己比│  │品比│  │場比│                           │
│  └────┘  └────┘  └────┘                        │
│                                                 │
│  品質三角形（每個洞察必備）                          │
│       數據                                       │
│       /\                                        │
│      /  \                                       │
│     /    \                                      │
│    /______\                                     │
│  So What   行動建議                               │
│                                                 │
└─────────────────────────────────────────────────┘
```

**簡單說：**
- 每個結論都要有數據支撐
- 看到異常就要找原因
- 不只說「是什麼」，還要說「為什麼」和「怎麼做」
- 每個指標都要跟自己、跟競品、跟市場比較

### 5. 報告品質 100 分制審核

每份報告都會經過 6 個維度、20 項檢查的品質審核：

| 維度 | 檢查什麼 | 扣分範例 |
|------|---------|---------|
| 結構完整性 | 有沒有缺章節？ | 缺 SWOT 分析 → 扣 5 分 |
| 數據品質 | 有沒有空話或缺數據？ | 「表現不錯」沒數字 → 扣 3 分 |
| 訪談覆蓋 | 客戶關心的都有回答嗎？ | 忽略客戶提的競品 → 扣 5 分 |
| 三層對比 | 自比/競比/環比都有嗎？ | 只有自比沒有競比 → 扣 4 分 |
| 數據一致性 | 報告跟原始數據吻合嗎？ | 數字跟 data.json 不一致 → 扣 5 分 |
| 產出完整性 | 截圖、圖表都有嗎？ | 缺少 Dashboard 截圖 → 扣 3 分 |

## 系統有哪些「技能」？

系統的能力是模組化的，每個技能（Skill）就像一本操作手冊：

| 技能 | 功能 |
|------|------|
| **orchestrator** | 管理者的工作流程 SOP |
| **data-analysis-methodology** | 16 章分析方法論（統計方法、解讀規則、禁止事項） |
| **data-extraction-engine** | 數據擷取引擎（DB 查詢、瀏覽器擷取） |
| **narrative-packaging** | 數據→故事的轉化規則 |
| **presentation-generator** | 報告產出（PPT / Slides / Docs） |
| **brand-analysis-looker-studio** | Looker Studio 操作 SOP |

**這些技能就是你可以幫忙優化的部分！**

## 你可以做什麼？

我們希望你加入後，主要負責以下工作：

### 1. 優化分析方法

目前的方法論是初版，需要根據實際分析經驗持續改進：
- 哪些統計方法在實務上更有效？
- 怎麼判斷數據異常的閾值？
- 如何更好地做品牌間的公平比較？
- 新的分析框架（例如最近從 Naver Study 學到的 Z-Score 標準化和 Misaligned Periods）

### 2. 調整技能（Skills）

技能就是 AI 的操作手冊。你可以：
- 修改分析流程和判斷規則
- 新增分析維度或檢查項目
- 調整報告的章節結構和呈現方式
- 改善品質審核的評分標準

**不需要寫程式**，技能檔案是 Markdown 格式的文字檔，用自然語言描述流程和規則就好。

### 3. 優化工作流

- 哪些分析步驟可以合併或省略？
- 訪談時應該問哪些關鍵問題？
- 報告的章節順序怎麼調整更好讀？
- 品質審核的標準需要調整嗎？
