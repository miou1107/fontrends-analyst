# Brand Analysis Workflow — 品牌分析工作流程

## Purpose

定義從用戶發起分析請求到產出結構化洞察的完整工作流程，涵蓋訪談、資料擷取、交叉驗證、14 維度分析。此流程由 Core 知識庫驅動，平台無關。

---

## Requirements

### Requirement: §0 分析前訪談

每次分析任務 MUST 從訪談開始，不得跳過。

#### Scenario: 完整訪談流程
- **GIVEN** 用戶啟動品牌分析
- **WHEN** Skill 進入訪談階段
- **THEN** 依序詢問 7 個問題（不一次全丟）：
  1. 報告給誰看？（受眾）
  2. 分析的主要目的？
  3. 報告的核心論點？（可跳過）
  4. 特別想知道或擔心被問到的問題？
  5. 行動建議的執行對象是誰？
  6. PPT 風格偏好？（配色/語言/頁數）
  7. 需要競品比較嗎？

#### Scenario: 訪談後確認
- **GIVEN** 7 個問題已收集完答案
- **WHEN** 訪談結束
- **THEN** 回一段確認話術：「這份報告的目標是 [目的]，給 [受眾] 看...」
- **AND** 等待用戶確認後才進入 §1

#### Scenario: 用戶要求跳過訪談
- **GIVEN** 用戶說「直接分析」或「不用問了」
- **WHEN** Skill 收到跳過指令
- **THEN** 使用預設值（受眾=品牌方、目的=品牌健康檢查、配色=品牌色、語言=繁中、長度=完整版 13 張）
- **AND** 告知用戶正在使用預設值，隨時可調整

### Requirement: §1 Dashboard 導覽

Skill MUST 識別 Looker Studio 頁面結構，不寫死任何 URL。

#### Scenario: 首次開啟 Dashboard
- **GIVEN** 用戶提供 Looker Studio URL
- **WHEN** Skill 在瀏覽器模式下開啟 URL
- **THEN** 識別左側導覽欄所有分頁名稱和 URL
- **AND** 記錄到當次分析的工作底稿

#### Scenario: 設定品牌篩選器
- **GIVEN** 已開啟 Dashboard 頁面
- **WHEN** 需要篩選特定品牌
- **THEN** 依照正確流程操作：
  1. 點擊主題篩選器
  2. 搜尋框輸入英文品牌名（如 `louis`，不是 `LV`）
  3. 勾選目標品牌 checkbox
  4. 清空搜尋框，輸入「台北101」
  5. 取消勾選台北101
  6. Escape 關閉
  7. 確認顯示正確品牌名
- **AND** 禁止使用「僅」按鈕（會跳回首頁）

#### Scenario: 每換頁重設篩選器
- **GIVEN** 已完成某頁資料擷取，要前往下一頁
- **WHEN** 切換到新頁面
- **THEN** 重新執行篩選器設定（Dashboard 限制：換頁會重置篩選器）
- **AND** 截圖確認篩選器狀態正確

#### Scenario: 非 Journey101 Dashboard
- **GIVEN** 用戶提供的 URL 不是 Journey101 Super Dashboard
- **WHEN** Skill 開啟 URL
- **THEN** 識別該 Dashboard 的實際頁面結構（不假設標準分頁）
- **AND** 識別可用的篩選器類型和選項
- **AND** 調整資料擷取策略

---

### Requirement: §2 資料擷取

Skill MUST 從每個 Dashboard 頁面提取結構化數據。

#### Scenario: 提取 KPI 卡片數據
- **GIVEN** 頁面上有影響力指數、發文數等 KPI 卡片
- **WHEN** 進行資料擷取
- **THEN** 使用截圖視覺讀取方式取得數值
- **AND** 記錄提取時間（Taipei time）和篩選品牌

#### Scenario: 提取圖表數據
- **GIVEN** 頁面上有 SVG 渲染的折線圖/柱狀圖
- **WHEN** 進行資料擷取
- **THEN** 截圖後視覺估算各數據點
- **AND** 對關鍵數據點 hover 取得 tooltip 精確值
- **AND** 標注估算值的誤差範圍

#### Scenario: 提取表格/排行數據
- **GIVEN** 頁面上有 KOL 排行或關鍵字列表
- **WHEN** 進行資料擷取
- **THEN** 優先使用 `read_page` 或 `get_page_text` 取得文字
- **AND** 失敗時改用截圖

#### Scenario: 資料記錄格式
- **GIVEN** 某頁面資料擷取完成
- **WHEN** 記錄數據
- **THEN** 使用標準格式：頁面名稱、提取時間、篩選品牌、各指標數值、備注

---

### Requirement: §3 交叉驗證

每個聲量高峰 MUST 有外部事件對應或標注為「待驗證」。

#### Scenario: 找到對應事件
- **GIVEN** 影響力指數在某月份出現高峰
- **WHEN** 系統搜尋「[品牌] [年月] campaign event news」
- **THEN** 找到對應事件（如代言人宣布、時裝週、新品發布）
- **AND** 在報告中標注事件名稱和來源

#### Scenario: 找不到對應事件
- **GIVEN** 影響力指數突增但找不到對應事件
- **WHEN** 系統檢查基準線是否同步拉高
- **THEN** 如果基準線拉高 → 標注「疑似數據源調整，非真實聲量變化」
- **AND** 如果基準線正常 → 標注「待驗證，建議人工核查」

#### Scenario: 競品驗證
- **GIVEN** 訪談時確認需要競品比較
- **WHEN** 主品牌資料擷取完成
- **THEN** 在同一 Dashboard 切換篩選器到競品
- **AND** 提取相同指標建立比較表

---

### Requirement: §4 14 維度分析

Skill MUST 對每個維度產出分析結果，無數據的維度標注原因。

#### Scenario: 執行核心維度（1-7）
- **GIVEN** 資料擷取和交叉驗證完成
- **WHEN** 進入分析階段
- **THEN** 依序執行 7 個核心維度：
  1. Dashboard 完整數據彙整
  2. 原始貼文質化分析（Top 5 開原文驗證）
  3. 外部交叉驗證結果
  4. 季節性 + 外部因素
  5. 品牌健康度
  6. 品牌與客戶場域關聯性
  7. 洞察回扣行動建議

#### Scenario: 執行進階維度（8-14）
- **GIVEN** 核心維度完成，且時間/資料允許
- **WHEN** 進入進階分析
- **THEN** 依可用性執行：YoY/MoM、競品、受眾、內容類型、時段、地理、品牌官網
- **AND** 無資料的維度標注「本次未提取」及原因

#### Scenario: 數據品質警示
- **GIVEN** 分析過程中發現數據品質問題
- **WHEN** 遇到語系偵測疑慮、情緒分析誤判風險、數據異常
- **THEN** 在分析結果中加入對應警示標注
- **AND** 這些警示會出現在最終報告的「數據品質聲明」頁
